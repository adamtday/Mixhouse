# Objection Matrix

## Purpose

Catalogue the strongest objections every audience will raise, codify the canonical response, identify the proof that substantiates it, and route each objection to the website surface that should preemptively neutralise it. The website itself is engineered as a progressive objection-neutralisation engine: a serious visitor should encounter their own strongest objection answered before they need to ask it.

## Key Takeaways

- Objections are predictable; ad-hoc answers are not. Pre-rehearsed responses win meetings and pre-empt loss of momentum.
- Every objection row has four attributes: response, proof needed, where on the site it resolves, and which supporting assets reinforce the rebuttal.
- The website is the objection-neutralisation engine. Every page brief in `08_website_strategy/page_briefs/` should be auditable against this matrix.
- Some objections are valid concessions, not rebuttals. Saying "you're partly right" is honesty currency in investor and broadcaster conversations. The Valid Concessions table at the end of this file makes the concessions explicit.
- The matrix expands when new objections surface in the wild. Adding a row is preferred over answering ad-hoc.

## Website Implications

The website is the objection-neutralisation engine. Audit each page brief against this matrix:

- `home.md` is responsible for pre-empting the fan-tier objections (F1, F2, F3, F4) through atmosphere, casting cues, and music previews.
- `the_show.md` is responsible for the format-confusion objections (S6, U10) and the talent-side objections (T1–T4 surfaced via the casting CTA flow).
- `the_world.md` is responsible for the cultural-credibility objections (AD1, T1, F1, U7) through brand world, advisor preview, and cultural posture.
- `business_model.md` is the spine that handles the investor / streamer / broadcaster / sponsor objections (I1–I8, S1–S7, BC1–BC4, SP1–SP4) at L3 Substance depth.
- `advisors.md` is responsible for AD1–AD4 plus carries trust signal to T1, F1, PR3, and SP-P1.
- `materials.md` (gated L4) is where the deeper-proof objections resolve (I5, I6, BC1, BC3, SP4, PR1, SP-P1, SP-P2, SP-P3).
- `contact.md` does not handle objections; it routes inquiries.

A page brief that does not pre-empt at least one objection from this matrix is unnecessary. Re-read this file before writing or revising any page brief.
