# Format Engine

## Purpose

Describe the recurring mechanic that makes MixHouse a *format*, not a *show*. The format engine is the reusable, franchisable, territory-portable structure that generates content, music, eliminations, and live events on a predictable cadence.

## Key Takeaways

- Every episode produces a finished, releasable track per active contestant or duo.
- Every episode ends in a live event (`Drop Night`) generating real audience signal.
- Eliminations are driven by a composite `Audience Index` (judges + crowd + streams + social), not a single panel vote.
- The engine compounds across the season: tracks released early continue to accrue streams that influence later eliminations.
- The same engine works in any territory — only the residency location, judges, and music scene change.

## Core Content

### The Weekly Loop

| Day | Stage | Output |
| --- | --- | --- |
| Mon | Brief + Crate Drop | Weekly creative theme set by judges; reference tracks added to `Crate`. |
| Tue | Studio sessions | Producers begin tracks. |
| Wed | Mentor Drop | Guest mentor reviews works-in-progress. |
| Thu | Mix + master | Tracks finalised and prepared for `Drop Night`. |
| Fri | `Drop Night` | Tracks debuted live to curated audience. |
| Sat | Release + reaction | Tracks pushed to streaming; short-form clips deployed. |
| Sun | Elimination + reset | `Audience Index` calculated; elimination filmed; new brief teased. |

This loop is week 1 of the season. Subsequent weeks reuse the same skeleton with escalating stakes, mentor caliber, and `Drop Night` venue scale.

### Composite Inputs

Each weekly elimination decision combines four signals:

1. **Judge weighting** — qualitative scoring against the weekly brief.
2. **`Drop Night` reaction** — quantitative crowd reaction (applause meter, dance-floor density, contestant-specific energy signal).
3. **Streaming performance** — first-72-hour streams across DSPs since release.
4. **Short-form virality** — TikTok/Reels velocity and qualifying engagement.

Detailed weighting in `02_show_bible/elimination_logic.md`.

### Format Portability

The engine is intentionally simple so it travels:

- Swap the residency location → new territory.
- Swap the resident judges → local scene credibility.
- Keep the loop, the crate concept, the `Drop Night` payoff, and the elimination logic.
- Local music partners adapt the release pipeline.

This is the same portability mechanic that made `Idol` and `The Voice` long-running franchises — but powered by music *creation* rather than audition.

### Format Engine vs Other Music Shows

| Format | What it produces each episode | What persists after the season |
| --- | --- | --- |
| `The Voice` | A performance of an existing song | An artist with a record deal (sometimes) |
| `Rhythm + Flow` | A freestyle / cypher | A breakout artist |
| `The Piano` | A live piano performance | A short-form viral moment |
| **MixHouse** | A finished, releasable track + a club moment | A growing catalogue of released music, a tour, a label deal |

The persistence column is the franchise asset. Every other column expires.

### The Cinematic Layer

The engine produces three filmable surfaces every week:

1. **Residency life** — character development, conflict, friendship, creative blockage. Love Island grammar.
2. **Studio process** — craft and creative decision-making. Drive to Survive grammar.
3. **Performance** — `Drop Night` and venue activations. Cercle/Boiler Room grammar.

A single episode interleaves all three. No previous DJ format has combined them.

### Failure Modes the Engine Guards Against

| Risk | How the engine mitigates |
| --- | --- |
| "Watching people stare at laptops." | Studio scenes are scored against character + craft and intercut with residency life and performance. |
| Predictable eliminations. | Composite `Audience Index` introduces real volatility week to week. |
| Music that doesn't land. | Mentor system and `Crate` keep quality floor high; release pipeline ensures finished masters. |
| Contestants peaking too early. | Streaming performance compounds — early hits keep contestants in even if a weak track follows. |

## Objections Answered

| Objection | Response |
| --- | --- |
| "Finishing a track every week is unrealistic for emerging producers." | Each contestant has access to studios, gear, mentors, and supervising engineers. The format is designed for finished, not perfect, tracks. |
| "Crowd reaction is unreliable." | It's one of four inputs, not the only one. The composite index smooths it. |
| "Streaming inputs introduce time lag." | First-72-hour data is available before each elimination film day. The lag is built into the schedule. |
